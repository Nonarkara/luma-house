# Luma House — deep audit, 2026-07-17

Method: full read of `src/` (App, plan, geometry, gestures, viewport, FloorPlan,
SpatialView, RenderGallery, Inspector, analysis engine, concept client, worker),
plus baseline `npm test` (64 green incl. 19 stale worktree duplicates), lint, build.
Every claim below was verified against the code line cited.

## CRITICAL / HIGH

1. **Concept-render success reported as failure when localStorage fills.**
   `src/concept/renderQuota.ts:30` `writeQuota` calls `localStorage.setItem` with no
   guard. The quota store keeps up to 12 base64 PNG data URLs; a single Gemini
   render is typically 1–3 MB, so the 5 MB browser quota is hit after ~2–4 images.
   When `setItem` throws, `recordConceptImage` (`generateConcept.ts:56`) propagates,
   so `runConceptRender` (`App.tsx:356`) shows "concept render failed" — after the
   server already consumed one of the user's 3 daily renders. Lose-lose.

2. **Climate "Apply" misplaces E/W windows.** `App.tsx:325` always writes
   `rotation: 0`. A window applied to a W or E wall renders horizontal on a vertical
   wall and `sunPatches` (`plan.ts:233-236`) computes its patch along the wrong axis.

3. **"Share" shares nothing.** `TopBar.tsx:31` copies the bare URL. Plan state lives
   only in localStorage, so a recipient sees *their own* plan (or the default), not
   the sender's. The button promises collaboration the architecture can't deliver.

4. **Quick action "brighter" plants a window off every room.** `App.tsx:369` hardcodes
   `(36, 6)` — 4% above the top wall of the demo plan (rooms start at y=10), outside
   `classifyOpening`'s 1.5% tolerance (`walls.ts:107`). The window is classified to no
   room: no sun patch, no ventilation credit, but the toast claims "Added a north
   window". Dishonest UI.

## MEDIUM

5. **Openings float.** `moveOpening` (`geometry.ts:98`) only clamps 1–99%. Windows and
   doors can sit mid-room or mid-site, breaking wall classification, sun patches and
   ventilation scores, and looking broken. No snap-to-wall on placement or drag.

6. **No delete for openings; zero keyboard support.** Rooms and furniture have delete
   affordances; openings have none. No Delete key, no Ctrl/Cmd+Z, no Esc, no arrow-key
   nudge. Resize handles are pointer-only (`aria-hidden`), so keyboard users can't
   resize either.

7. **Room overlap unguarded.** Nothing stops two rooms overlapping; `totalArea`
   double-counts the overlap, silently breaking the "100 m² is the cost anchor" claim
   in budget mode and distorting shared-wall detection in `exteriorWalls`.

8. **Export without import.** `exportPlan` writes a project JSON nothing can read
   back. `readSavedPlan` also trusts parsed JSON shape blindly (`App.tsx:26`) — a
   malformed payload crashes at render.

9. **Vitest runs stale `.worktrees/**` tests.** 19 duplicate tests from the old
   `feature/environmental-analysis-and-photo-input` worktree run on every `npm test`.

10. **`estimateEnergySavings` duplicated.** Inspector.tsx:453 re-implements the formula
    inline instead of importing from plan.ts — the two will drift.

11. **Canonical/OG URLs point to the GitHub Pages mirror**, not the primary
    luma-house.pages.dev (index.html:8,14,15).
